OSS PreflightProblem
Problem

Developers lose time choosing a stack before they write useful code.

  • Package choice is often driven by memory, popularity, or search ranking.
  • Facts, inferred tradeoffs, and stale assumptions get mixed together.
  • Even a good choice still leaves starter setup and verification work.
OSS PreflightProblem
Solution

A single honest pipeline: idea -> evidence -> scaffold -> smoke test.

Recommend Parse the idea, collect public package facts, and rank candidates deterministically.
Explain Show score components and separate registry facts from AI-generated inference.
Generate Create a starter project with mocked messages and a runnable smoke test.
